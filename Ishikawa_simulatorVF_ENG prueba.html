Phase 3 · Cause & Effect (Ishikawa) Diagram

Build the fishbone diagram

The causes you sorted in Phase 2 are now placed on the diagram. Edit, move, add, or remove causes as needed — then move on to the next priority defect.

3-level structure: L1 · Cause → the main cause branch attached to the bone (e.g. "Machine overheats") · L2 · Sub-cause → why that cause happens (e.g. "Technician is slow") · L3 · Root cause ★ → the deepest reason, shown in red (e.g. "Only one technician"). Use the edit panel below to add levels.

Interactions: drag any label to reposition it (the connecting arrow follows). Click once to select (orange), then click another bone to move it there. Double-click to edit text. Drag the small circle at a bone's base to slide the bone left/right.
Phase 4 · The 5 Whys

Drill down to the root cause

Finding a cause is not the same as finding the root cause. For each cause on your fishbone diagram, ask "Why?" up to five times until you reach something that, if fixed, would prevent the problem from recurring.

Worked example

CauseWorn cutting tool (Machine)
Why? (1)It is not replaced on time
Why? (2)There is no preventive maintenance plan
Why? (3)Tool life is not monitored
Why? (4)No indicator exists for tool wear
Why? (5)Maintenance works reactively, not proactively
🎯 Candidate root cause
Phase 5 · Validation

Validate your candidate root causes

Finding a root cause is not the same as proving it. For each candidate root cause, answer the 5 validation questions — the more "Yes" answers, the stronger the evidence that this is the true root cause.

Prioritize corrective actions

For each validated root cause, propose a corrective action and rate its impact and effort. Quick Wins (high impact, low effort) should usually go first.
