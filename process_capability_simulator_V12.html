What is SPC?
Definition · purpose · what a control chart does
Definition

Statistical Process Control (SPC)

SPC uses statistical methods to monitor a process over time and detect special-cause variation — non-random changes that push a process out of stable behavior. The main tool is the control chart, which plots a quality characteristic with statistically derived limits that signal when the process behaves unexpectedly.

②
Variable vs. Attribute data
The most fundamental distinction — determines which chart you use
📏 Variable Data

Measurable, continuous quantities

Values that can take any number along a scale. You measure them with a device (ruler, scale, thermometer). Rich in information — one measurement carries a lot of quality signal.

Examples: diameter (mm), weight (g), temperature (°C), tensile strength (MPa), fill volume (mL), cycle time (s).
X̄ Chart R Chart → Use for Cp, Cpk
✓✗ Attribute Data

Counted, categorical results

Values that result from classifying or counting. Either a unit passes or fails, or you count the number of defects. Less information per data point — you need larger samples.

Examples: number of scratches per panel, % defective bottles, count of mislabeled packages, missing welds per assembly.
p / np Chart c / u Chart
Key rule: If you can say "how much" or "how many mm", it is variable data → use X̄-R chart. If you can only say "yes/no", "defective/good", or "count of defects", it is attribute data → use p, np, c, or u chart. The chart type determines which formulas apply — and Cp/Cpk only apply to variable data.
③
Common vs. Special causes of variation
Understanding why processes go out of control — and what to do about it

✅ Common Cause Variation

Random, inherent noise in every process. Stable and predictable within statistical limits. Cannot be attributed to one specific cause.

  • Machine vibration within tolerance
  • Small differences in raw materials
  • Normal operator variation
  • Environmental fluctuations

Process is in control. Points randomly distributed within limits. No action needed — improvement requires redesigning the system.

⚠ Special Cause Variation

Non-random, assignable to a specific identifiable event. Signals that something has changed. Must be investigated and eliminated.

  • Broken or worn tool
  • New operator or shift change
  • Different supplier or material lot
  • Equipment malfunction or miscalibration

Process is out of control. A Western Electric rule is violated. Find and remove the cause before assessing capability.

⚡ The most important distinction in SPC

This distinction applies to both variable and attribute data — but the formulas differ. The columns below show the variable-data formulas (X̄-R); for attribute charts (p, np, c, u) the control limits use ±3 standard deviations of the binomial/Poisson distribution instead of A₂R̄.

Spec limits (red) — Voice of the Customer
Variable data: LSL, USL on individual measurements → used in Cp, Cpk, Zu, Zl

Attribute data: there are usually no spec limits — you compare the defect rate to a target (e.g. AQL).
Set externally — the process has no say in these. Cp/Cpk apply ONLY to variable data.
Control limits (blue) — Voice of the Process
Variable (X̄): UCL = X̄̄ + A₂R̄, LCL = X̄̄ − A₂R̄

Attribute (e.g. np): UCL = np̄ + 3√(np̄(1−p̄)), LCL = max(0, …)

Both are computed FROM the process data itself.
Detect instability. Never set equal to spec limits. Apply to both data types.

A process can be in statistical control and still produce defectives. For variable data, that is exactly what Cp < 1 means: the process is stable, but its bell curve is wider than the spec window. For attribute data, a stable (in-control) defect rate can still be higher than the customer will accept.

Western Electric rules — detecting special causes
1
One point beyond ±3σ (UCL or LCL) — the most basic rule. Any single point outside the control limits. Indicates a sudden large shift in the process.
2
8 (or 9) consecutive points on one side of the center line — a sustained shift in the mean. Statistically unlikely under randomness even though no point exceeds the limits.
3
6 consecutive points steadily increasing or decreasing — a trend. Suggests gradual change such as tool wear, temperature drift, or material degradation.
4
14 alternating points up and down — systematic oscillation. Often indicates mixing of two process streams or measurement system issues.
SPC workflow

📋 Step 1 — Collect & group data

Collect individual measurements and organize into rational subgroups (usually 4–5 units per subgroup — one "snapshot" of the process). Minimum 20–25 subgroups for reliable limit calculation.

📐 Step 2 — Calculate control limits

Compute X̄̄ and R̄. Calculate UCL/LCL using A₂, D₃, D₄ constants (depend on n). Never set limits equal to spec limits.

📈 Step 3 — Plot and check rules

Plot X̄ and R charts. Apply the Western Electric rules. If any rule is violated, identify and remove the special cause before continuing.

🔬 Step 4 — Assess capability

Only after the process is in statistical control does capability analysis make sense. Compute Cp, Cpk (short-term) and Pp, Ppk (long-term). Compare to the 1.67 target.

1. A technician measures the diameter of each bolt produced and records the value in millimeters. What type of data is this?
a) Attribute — it classifies the bolt as good or defective
b) Variable — it is a measurable, continuous quantity
c) It depends on how many bolts are in each sample
Correct. Diameter is a measurable quantity → variable data → X̄-R chart. If you only recorded "pass / fail", it would become attribute data.
2. You inspect batches of 50 bottles and count how many are defective in each batch. All batches have exactly 50 bottles. Which chart should you use?
a) X̄-R chart
b) p chart
c) np chart — counts of defectives with constant n
Correct. Defective items + constant n = np chart. If n varied per batch, you would use a p chart (proportion) instead.
3. A process has Cp = 1.5 and Cpk = 0.6. What is the best description of the situation?
a) The process is too variable — σ̂ must be reduced
b) The process is narrow enough but the mean is off-center — centering is the problem
c) The spec limits are too narrow
Correct. Cp = 1.5 means Q fits within the spec window (good spread). But Cpk = 0.6 < 1 means the mean is close to one spec limit. Shifting μ toward center would fix this without changing σ̂.
4. Control limits on an X̄ chart are set at X̄̄ ± A₂R̄. What is true about these limits?
a) They are the same as the specification limits (LSL and USL)
b) They apply to individual unit measurements
c) They apply to subgroup means and are narrower than spec limits because of the ÷√n effect
Correct. Control limits apply to X̄ (subgroup means), not individual units. Because means vary less than individuals, UCL/LCL are much narrower than spec limits. Never set them equal — that is a classic SPC mistake.
5. All 25 points on an X̄ chart fall within the UCL and LCL. Can we conclude the process is fully capable of meeting the spec limits?
a) Yes — if the process is in control, it must be meeting spec
b) No — in-control means the process is stable, but it could still produce defectives if Cp < 1
c) Only if Cpk > 0
Correct. "In control" (statistical stability) and "capable" (meeting spec) are completely different concepts. A process with Cp = 0.6 can be perfectly in control (all means within UCL/LCL) while individual units regularly fall outside LSL or USL.

📐 When do we use σ̂ and when do we use S?

σ̂ = √variance — for capability (Cp/Cpk)
σ̂ = √S² = √[Σ(xᵢ−X̄̄)²/(N−1)], the standard deviation of all individual values (matches Excel STDEV). This is what we use for Cp and Cpk.
Note: control charts use a different estimate, R̄/d₂, for their limits — see the Control Charts tab.
S — long-term (total variation)
Used for Pp and Ppk.
Includes drift, shifts, and between-subgroup variation accumulated over time.
Represents what the customer actually experiences over the long run.

Rule of thumb: if you only have one short data collection → use σ̂ (Cp/Cpk). If you have data spanning many shifts/days/lots → the long-term S gives Pp/Ppk. When the process is perfectly stable, σ̂ ≈ S and Cp ≈ Pp.

❓ "If the process is inside the spec limits but outside the control limits, why are there still defectives?"

Great question — and it reveals the key confusion. Control limits and spec limits answer different questions and apply to different things:

  • Control limits apply to subgroup means (X̄) and tell you if the process is stable. A point outside them = a special cause has shifted the process. This is about predictability over time.
  • Spec limits apply to individual units and tell you if a part is acceptable. A unit outside them = a defective part. This is about meeting the customer requirement.

So a process can be out of control (unstable — means jumping around) yet most individual parts still fall inside spec. OR it can be perfectly in control (stable) yet produce defectives because its natural spread (Q = 6σ̂) is wider than the spec window (Cp < 1). Stability ≠ meeting spec. You need BOTH: a stable process (in control) AND a capable one (Cp, Cpk adequate).

⚡ Two different questions — Variable data only

⚠ This box applies to variable (measurable) data — X̄-R charts. Attribute charts (p, np, c, u) have different logic: they do not use spec limits, Cp/Cpk, or the normal distribution framework.

Question 1: Is the process stable? → Answered by the control chart. If all subgroup means (X̄) fall within UCL and LCL, the process is in statistical control — it behaves predictably over time.

Question 2: Is the process capable? → Answered by Cp and Cpk. Even if the process is stable, the bell curve of individual measurements might be wider than the spec window — meaning it produces defectives even while being "in control."

✏️ Quick quiz — Process Capability
1. A process has Cp = 1.8 and Cpk = 0.7. What is the primary problem?
a) The process is too variable (σ̂ is too large)
b) The process is off-center (mean is not near the nominal)
c) Both Cp and Cpk are too low
Correct. Cp = 1.8 means the bell curve easily fits the spec window (good spread). But Cpk = 0.7 < 1 because the mean is too close to one limit. The solution is to center the process — no need to reduce variability.
2. If Cp = 0.75, what is the FIRST action?
a) Adjust the process mean toward the center
b) Reduce σ̂ (reduce process variability)
c) Widen the spec limits
Correct. Cp < 1 means Q = 6σ̂ is wider than the spec window. Even perfectly centered, defects are inevitable. The only fix is to reduce σ̂. Centering alone can never fix a Cp < 1 situation.
3. How is σ̂ calculated from sample data? (As taught in this course)
a) σ̂ = R̄/d₂ (from the control chart)
b) σ̂ = √(S²) = √[Σ(xᵢ − X̄̄)² / (N−1)] (matches Excel STDEV)
c) σ̂ = Q / 6
Correct. For capability (Cp/Cpk) in this course, σ̂ is the sample standard deviation of all individual values — the same as Excel's STDEV. Note: R̄/d₂ is also a valid estimator, but we reserve it specifically for building control chart limits (within-subgroup variation). Two estimators, two purposes — don't mix them up.
4. A process has Q = 3.2 mm and the spec window (USL − LSL) = 4.0 mm. The mean is perfectly centered. What is Cp and is the process capable?
a) Cp = 4.0/3.2 ≈ 1.25 → Marginally capable (needs control, 1.0–1.33 range)
b) Cp = 3.2/4.0 = 0.8 → Not capable
c) Cp = 1.0 → Exactly at the boundary
Correct. Cp = (USL−LSL) / 6σ̂ = (USL−LSL) / Q = 4.0/3.2 = 1.25. This falls in the "adequate but needs control" range (1.00–1.33). Since centered, Cpk = Cp = 1.25. The process just barely meets requirements.
5. What does it mean if Pp < Cp for the same process?
a) The process is not centered
b) There is a calculation error
c) The process drifts over time — long-term variation is greater than within-subgroup variation
Correct. Cp uses short-term σ̂ (within-subgroup variation only). Pp uses long-term S (total variation including drift). If Pp < Cp, the process is not as stable over time as it appears in short runs — there is between-subgroup drift (shift-to-shift, lot-to-lot, etc.).
Why is Cp usually greater than Cpk?

Cp ignores where the mean is — it only asks "is the bell narrow enough to fit the spec window?" Cpk also considers centering — it measures the distance from the mean to the nearer spec limit. When the mean is off-center, one side is closer, so Cpk drops below Cp. They are equal only when the process is perfectly centered.

Pp and Ppk — long-term performance (advanced)
⚠ Didactic simulation of long-term variation
In a real study, Pp and Ppk are calculated from actual long-term data collected over many shifts, days, or lots. Here we don't have that data, so we simulate long-term drift with a multiplier — this is a teaching aid to show the concept, not how you compute Pp/Ppk in practice. In practice: S (long-term) comes from real accumulated data, not from scaling σ̂.
What is the stability multiplier?
In the short term, the process has a natural spread measured by σ̂ (the √variance of the data). But over days, shifts, or material lots, the process may drift — its overall spread grows. The stability multiplier simulates this drift: it scales σ̂ upward to represent what the process looks like long-term. Pp and Ppk are then computed with this larger σ (called S, the long-term standard deviation), to illustrate what the customer experiences over time vs. what the process could do in a single short run.
Stability multiplier — σ_long = σ̂ × mult
1.0 = perfectly stable (Pp = Cp). >1 = drift or shift-to-shift variation present.
